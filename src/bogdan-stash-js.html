Самое важное в JavaScript - выражения, функции, объекты.

Практически все сущности в JavaScript - объекты.

Объект

это набор свойств "имя: значение". Имя ещё называют ключами.

  • Массив - это объект
  • Функция - это объект
  • Число и строка это примитивные значения и они ведут себя как объекты

Метод - это функция которая являеться значением свойства объекта.

  • console - объект.
  • . - это точечная запись. С помощью точечной записи получаем доступ к методу объекта.
  • log - метод.
  • ( ) - вызов метода log.
  • 'Hello World' - аргумент. Значение типа 'string'.

Любое выражение всегда возвращает значение

    Примеры выражений и их результаты:
  • 'abc' - 'abc'
  • 10 - 10
  • 5 + 2 - 7
  • c = 10 - вернёт 10
  • 'Good ' + 'Evening' - 'Good Evening'
  • a <= b || c !=d - true/false
  • myFunction(c, d) - результат функции

Переменные дают возможность повторного доступа к значениям.

    Как правильно называть переменные:
  • PascalCase

    Используется для именования типов и классов в JS;

  • DB_PASSWORD

    Используется для именования переменной, значение которой известно до запуска приложения и не меняется (const);

  • сamelCase

    Используется во всех остальных случаях, помимо вышеуказанных.

    Объявление переменных
  • let

    Можно переназначить, переприсвоить. Можно не присваивать значение, в таком случае значение будет undefined;

  • const

    Нельзя переназначить. Обязательно сразу присвоить значение.

  • var

    Ключевое слово до ES6

    Примитивные типы
  • string (строка)
  • boolean (логический)
  • number (число)
  • null
  • undefined
  • symbol (символ)

При присваивании значения такого типа, в памяти компьютера эти переменные будут содержать значения.

    Ссылочный тип
  • object (объект)

Переменная содержит ссылку на область в памяти где находится объект.

  1. Объявляется переменная objectA со значением object
  2. В памяти будет создана переменная с названием objectA
  3. Объект будет записан в другом месте памяти
  4. В переменную objectA будет записана ссылка на объект

В данном примере копируется ссылка на объект objectA

В результате в памяти компьютера будут находится 2 переменные, которые будут ссылаться на 1 объект.

При изменении значения свойства изменится объект. т.е значение поменяется перейдя по ссылке. (добавление свойства работает аналогично)

JavaScript - динамически типизируемый язык и позволяет объявлять функции, объекты и переменные без объявления типа.

Стараться использовать const везде, где это возможно.

JSON

JSON (Javascript object notation)

Формат обмена данными.

JSON передаётся в виде строки.

    Конвертация JS объекта в JSON
  • JSON.stringify(аргумент)
    Конвертация JSON в JS объект
  • JSON.parse(аргумент)

Копирование примитивных типов происходит по значению (copy by value)

Копирование ссылочного типа (copy by reference) - меняется исходный объект

    Как избежать мутаций (создание копий объекта и изменение копии, не меняя оригинальный объект)
  1. Метод assign

    Создаст новый объект - {}, и запишет все свойства объекта user. НО если у объекта есть вложенные объекты, то при создании нового объекта ссылки на них сохраняются.

  2. Spread (Оператор разделения объекта на свойства)

    Объект разделяется на свойства и сразу же собираются в новый объект который в {}. НО если у объекта есть вложенные объекты, то при создании нового объекта ссылки на них сохраняются. Аналог assign, но более удобнее и короче.

  3. JSON.parse - JSON.stringify

    JSON.stringify конвертирует объект в строку, затем JSON.parse конвертирует обратно в объект. В результате получается полностью новый объект.

Функция

Блок кода, который можно выполнять многократно.

    Функция может быть:
  1. Именованной
  2. Присвоенна переменной
  3. Анонимной
  4. Аргументом при вызове другой функции
  5. Значение свойства объекта (методом)
Функция

это объект.

Структура функции

  • function

    ключевое слово

  • myFn

    имя функции

  • a,b

    параметры, ведут себя как переменные. Значения определяются в момент вызова функции.

  • {...}

    тело функции

  • return

    возвращает результат. После return функция прекращает выполнение. Если нет return, то возвращает undefined

  • myFn(10, 8)

    вызов функции с аргументами 10 и 8

Внутри функции можно изменять внешний объект, он будет изменятся по ссылке, но делать это крайне не рекомендуется.

Если же необходимо изменять объект внутри функции, лучшим решением будет создать его копию внутри функции.

Колбэк функции

Колбэк функция

это функция которая вызывается внутри другой функции.

У функции callFunctions параметр колбэк функция. т.е в теле 'callFunctions' вызывается колбэк функция.

В данном примере встроенная функция setTimeout вызывает callback-функцию printMyName.

Правила работы с функциями

  1. Называть функцию исходя из выполняемых задач
  2. Одна функция должна выполнять одну задачу
  3. Не рекомендуется изменять внешние относительно функции переменные

Область видимости определяет границы действия переменной

Типы областей видимости

  1. Глобальная область видимости
  2. Область видимости функции
  3. Область видимости блока (любой код между { } )

Пример как делать не рекомендуется

  • Объявлена функция myFn, внутри которой присваивается значение true переменной 'a'
  • Переменной 'a' нету ни в области видимости функции, ни в глобальной области видимости (попытка присвоить значение не объявленой переменной)
  • Переменная 'a' будет автоматически объявлена не в области видимости функции, а в глобальной области видимости

Не рекомендуется присваивать значение переменным которые небыли объявлены ранее.

Строгий режим (strict mode)

Инструкция интерпретатору JS анализировать код более пристально

'use strict'

строка должна быть первой в глобальной области видимости или в области видимости функции

Например, если 'use strict' активен, то переменные не будут автоматически объявлятся.
